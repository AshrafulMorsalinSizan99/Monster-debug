Question: What are the differences between local storage and session storage?

Ans:
i.Local Storage is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser wheareas the sessionStorage is very similar to the localStorage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab or session is on. Once you close the tab or terminate the session, the data on session storage also gets lost.

ii.As local storage is not session-based, it must be deleted via javascript or manually.But session storage is session-based and works per window or tab. This means that data is stored only for the duration of a session, i.e., until the browser (or tab) is closed

iii.The storage capacity of local storage is 5MB/10MB whereas the storage capacity of session storage is 5MB

iv.There are fewer old browsers that support it

Question: What are the difference between global scope and block scope?

Ans:

i. The variables defined outside of any function or curly brackets are known as global variables and have global scope.Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {}.

ii.Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable whaereas in block scope, the variables can be accessed in fixed part of the program.

Question: How does javascript event loop work?

Ans: The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.
